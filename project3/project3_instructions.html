PROJECT 3: EXTERNAL CSS (MUSICIAN WEBSITE)
 OBJECTIVE
The purpose of this project is to demonstrate your understanding of the following course outcomes:

1. Gather and analyze requirements to design and plan a website.
2. Produce professional websites that adhere to appropriate/established standards, including usability, accessibility, and validation.
3. Select and apply the necessary tools, protocols, and resources to support web technologies.
4. Select comply with legal precedent and ethical standards when constructing websites.


The home page of Tyler Kealey's music website. https://tylerkealey.com/music.

PROJECT DESCRIPTION
Project 3 requires you to create a Music Artist Website for a real band, group, or individual MUSIC artist of your choice. We don't have enough time in the class for you to create a complete Music Artist Website, so will create three separate HTML5 webpages (Home, Music, and News). For this website, you will have to use an External Style Sheet.  An External Style Sheet is a separate file, designated by a .CSS file extension, that is linked to a web page.  All the styles used on a website can be declared in the External Style Sheet, making it the most efficient way to make global changes to a website.  In this project, you will design a three (3) page website that utilizes an External Style Sheet. This must be a completely new site and not a modification of what you submitted for Project 1. Each page should have an appropriate name ending in .html. (index.html, music.html, and news.html). These pages must be linked together using navigation that is easy to find and use. IN ORDER FOR THE EXTERNAL STYLE SHEET TO WORK, IT MUST BE REFERENCED IN THE HTML PAGES YOU CREATE. Make sure all of the page requirements listed below are covered.
In addition to hosting your 3-page website on the ALOFT Web Server, you also need to submit ALL your project files to the appropriate Assignments Folder in our LEO classroom. I'll be viewing your work on the web and grading based upon what I see on the web. The URL you submit should work and your pages must be viewable (no problems with permissions). Questions? Please ask *before* turning in the assignment.
*** YOUR WEBSITE MUST BE ORIGINAL AND UNIQUE ***

 STEP 02: One your computer (locally), create an external CSS file and save it as style.css inside the project3 folder. REMINDER: You have already created the directory structure when you completed Project 1.  
STEP 03: LINK your CSS file to your webpages. In order for the External CSS file to work, it has to be referenced in the HTML5 pages (see example below):

PROJECT REQUIREMENTS
PAGE REQUIREMENTS
01: HOME PAGE (index.html)
a. The Home page must include the name an image of the artist.
b. The Home page must include background information about the artist.
02: MUSIC (music.html)
a. The Music (Discography) page must include at least three albums that the artist has released.
b. Each album must include information about each album (album cover, album name, list of songs, and publishing date.
03: NEWS (news.html)
a. The News page must include the latest news about the artist. This news must be REAL NEWS (you can do a google search to find the latest news about the artist).
04: EACH PAGE
a. Each page must utilize the CSS External Style Sheet.
b. Each page must include navigation.
SITE MAP

HTML REQUIREMENTS
01: PROPER HTML5 PAGE STRUCTURE & SYNTAX: Your web page must follow proper HTML5 page structure as listed by w3Schools.
02: VALIDATION: Your web page AND CSS should validate WITHOUT ERRORS using the W3c HTML validator at http://validator.w3.org and W3c CSS validator at http://jigsaw.w3.org/css-validator/.
How to do this:
Once you have uploaded your web files to the ALOFT Web Server, go to the w3c HTML validator and paste in the URL to your index.html page. Click the Check button. If there are errors, fix them. If there are no errors, copy the URL of the results page and include it in your Project Reflection (Item 3 under Submission Requirements below). Repeat the process for your other pages. Next go to the W3c CSS validator. Provide the path to assignment.css. If your page validates without errors, you'll be given embed code. Include this information in your Project Reflection (Item 3 under Submission Requirements below).
03: NAMING CONVENTION: Save/Name your webpages as index.html, music.html, news.html, and your External CSS file as style.css.
04: Add the following styles to your External CSS file:
a. H1 Heading Style (color, font family, center alignment, and font size).
b. H2 Heading Style (color, font family, center alignment, and font size).
c. Paragraph Style (color, font-family, and font-size, margin).
d. Create at least one ID Attribute.
e. Create at least one CLASS Attribute.
f. Body (Add a background color and text color that is easy to read together. Set the font family to Verdana with fallbacks to Arial and sans-serif).
g. Page Header and Footer (Create styles for both).
h. Wrapper or Content Container (Create a Wrapper or Content Container that keeps your content from bumping into the sizes of the web browser).
i. Navigation (Create a style to Center your navigation). Your navigation should include links to all three pages of your fan website.
05: IMAGES: Cite your source in your image caption. Optimize your images for the web so that their file sizes are 100k or lower. Wrap text around your images using CSS floats IF your design needs text wrap. Include padding so the text doesn’t bump up against the edges of your images. Clear your floats. Make at least one of your images a hyperlinked image. That means that you can click on the image and something happens (you go to a larger version of the image, you go to a webpage, you go to an e-mail link, etc.).
